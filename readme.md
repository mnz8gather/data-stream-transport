### 场景对比

- SSE(HTTP/1.1)
  - 简单，单向推送，适合通知。
  - 依赖 text/event-stream 和长连接。
- WebSocket
  - 双向通信，灵活，适合交互场景。
  - 手动实现复杂，生产环境推荐 ws.
- HTTP/1.1 流(Chunked)
  - 通用，适合流式 API。
  - 需要手动解析，效率较低。
- HTTP/2
  - 高效，支持多路复用，内置流机制。
  - 需 HTTP/2 兼容客户端和服务端。
- gRPC
  - 高性能，强类型，适合微服务。
  - 依赖 Protobuf 和 HTTP/2，浏览器支持需额外配置。
